VACUUM FULL

VACUUM FULL não é recomendado para versões do PostgreSQL anteriores à 9.0, exceto em circunstâncias extremas.

VACUUM FULL deve ser usado apenas se grandes quantidades de dados foram atualizadas ou excluídas em uma tabela, para liberar espaço de volta para o sistema operacional. Nos casos em que se espera que o espaço seja usado pela tabela no futuro, um VACUUM regular deve ser usado.

VACUUM FULL colocará bloqueios exclusivos no banco de dados, impedindo toda outra atividade na(s) tabela(s) sendo submetida(s) a VACUUM, e como tal terá um impacto muito grande em outros usuários do banco de dados enquanto é executado.

Após uma tabela ou banco de dados ter passado por VACUUM FULL, é recomendado executar REINDEX em todos os índices da tabela também.
