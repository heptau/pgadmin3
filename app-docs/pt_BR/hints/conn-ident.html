Falha na autenticação Ident

O servidor não aceita o usuário atual: o servidor informa

Se esta mensagem aparecer, a entrada pg_hba.conf encontrada para sua combinação cliente / usuário / banco de dados está configurada para autenticação "ident". Algumas distribuições, por exemplo Debian, têm isso como padrão. Para realizar a autenticação baseada em ident com sucesso, você precisa de configuração adicional; consulte a ajuda do PostgreSQL para obter mais informações. Para um iniciante, pode ser mais apropriado usar um método de autenticação diferente; senhas criptografadas MD5 são uma boa escolha, que pode ser configurada por uma entrada no pg_hba.conf como esta:

host all all 192.168.0.0/24 md5

Este exemplo concede acesso com senha criptografada MD5 a todos os bancos de dados para todos os usuários na rede privada 192.168.0.0/24.

Você pode usar o editor pg_hba.conf integrado ao pgAdmin III para editar o arquivo de configuração pg_hba.conf. Após alterar o pg_hba.conf, você precisa acionar um recarregamento da configuração do servidor usando pg_ctl ou parando e reiniciando o processo do servidor.
