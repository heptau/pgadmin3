Recomenda-se ativar o autovacuum

Introduzido no PostgreSQL 8.1, o processo autovacuum, que anteriormente era implementado como um serviço externo, agora está integrado ao servidor. Ele executará o VACUUM automaticamente se uma quantidade configurável de dados tiver sido alterada.

Executar o VACUUM é obrigatório em um servidor PostgreSQL para manter os dados consistentes e o servidor com bom desempenho. Para começar, é recomendável ativar o daemon autovacuum definindo 'autovacuum', 'stats_start_collector' e 'stats_row_level' em postgresql.conf como 'on'.

No PostgreSQL 8.3 e versões mais recentes, o autovacuum está ativado por padrão, a menos que você desative 'track_activities' ou 'track_counts'.

Com uma penalidade de desempenho geralmente insignificante do rastreamento de alterações de dados, você ganhará execuções automáticas de VACUUM.

Para ajustar o desempenho do autovacuum para necessidades individuais, talvez seja necessário ajustar suas configurações usando parâmetros globais em postgresql.conf ou individualmente em cada tabela; consulte a documentação.
