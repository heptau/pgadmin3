Instrumentação do servidor

O servidor não possui funções de instrumentação.

O pgAdmin III usa algumas funções de suporte que não estão disponíveis por padrão em todas as versões do PostgreSQL. Estas permitem algumas tarefas que facilitam a vida ao lidar com arquivos de log e arquivos de configuração.

O adminpack é instalado e ativado por padrão se você estiver usando a distribuição 'oficial' do pgInstaller do PostgreSQL para Windows e está incluído como um módulo contrib em todas as versões do PostgreSQL 8.2 e superiores. No entanto, se você estiver executando qualquer outra versão do PostgreSQL, precisará instalá-lo manualmente. Para fazer isso, basta copiar o código fonte apropriado do adminpack para o diretório /contrib da sua árvore de código fonte do PostgreSQL pré-configurada. Você pode baixar os adminpacks aqui.

Execute os seguintes comandos (substituindo admin por admin81 se necessário):

    # cd $PGSRC/contrib/admin
    # make all
    # make install

O módulo agora deve estar compilado e instalado em sua instalação existente do PostgreSQL. Para ser usado, você deve agora instalar o módulo no banco de dados que você usa como Banco de Manutenção do pgAdmin (normalmente postgres, embora você possa usar um banco de dados diferente). Para fazer isso, localize o arquivo admin.sql ou admin81.sql instalado na etapa anterior - normalmente pode ser encontrado em /usr/local/pgsql/share/contrib/ e execute-o contra seu banco de dados. Normalmente você precisa fazer isso como superusuário do PostgreSQL:

    $ psql postgres < /usr/local/pgsql/share/contrib/admin.sql

Na próxima vez que você se conectar ao servidor com o pgAdmin, ele detectará a presença do adminpack e oferecerá funcionalidade completa.

A instrumentação estendida do servidor não é suportada para PostgreSQL 7.3 e 7.4.
