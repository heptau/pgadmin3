Editando Visões, Procedimentos Armazenados ou Funções

Ao editar o código de uma visão, procedimento armazenado ou função, em vez de usar a pequena caixa de texto no diálogo de propriedades do objeto, você pode achar mais conveniente usar as opções de script no pgAdmin para editar o objeto na ferramenta de consulta. Isso lhe dá os benefícios dos recursos do editor da Ferramenta de Consulta, como autocompletar, localizar e substituir e uma área de trabalho maior.

Para usar as opções de script, na janela do navegador pgAdmin, clique com o botão direito no objeto que deseja editar e selecione a opção Script CREATE. Ela pode estar no submenu Scripts. Alternativamente, você pode selecionar o objeto e depois escolher a opção Script CREATE no submenu Scripts do menu Ferramentas.

Edite a definição do objeto conforme necessário e use o botão Executar Consulta para aplicar suas alterações. Não se esqueça de que modificar o nome de uma visão ou função, ou a assinatura de uma função ou procedimento, pode resultar na criação de um novo objeto junto ao original. Este efeito colateral útil torna conveniente criar objetos novos, mas semelhantes. Talvez seja necessário atualizar o navegador pgAdmin para exibir os objetos recém- criados.

Você pode usar esta técnica para realizar todos os tipos de operações em diferentes objetos, no entanto a sintaxe CREATE OR REPLACE do PostgreSQL a torna particularmente útil para editar visões, procedimentos armazenados e funções.
