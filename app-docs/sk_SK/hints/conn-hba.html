Prístup k databáze zamietnutý

Server nepovolil prístup do databázy: server hlási

Aby ste mali prístup k databáze na serveri PostgreSQL, musíte mať v prvom rade základný prístup k serveru pre vášho klienta (Host Based Authentication). PostgreSQL skontroluje, či súbor pg_hba.conf obsahuje záznam zodpovedajúci vašej kombinácii adresa klienta / meno používateľa / databáza a povolí prístup predtým, než sa vykoná vyhodnotenie prístupových oprávnení daných SQL GRANT.

Nastavenia v pg_hba.conf po inštalácii sú dosť obmedzujúce, aby sa zabránilo bezpečnostným dieram spôsobených neprehodnoteným a pritom povinným systémovým nastavením. Pravdepodobne budete chcieť doplniť niečo takéhoto

host all all 192.168.0.0/24 md5

Tento príklad udeľuje na základe hesla šifrovaného pomocou MD5 prístup ku všetkým databázam pre všetkých používateľov v privátnej sieti 192.168.0.0/24.

Na úpravu súboru pg_hba.conf môžete použiť editor nastavení, ktorý je súčasťou pgAdmin III. Po vykonaní zmien v pg_hba.conf je potrebné prinútiť server, aby nové nastavenie načítal. Buď pomocou pg_ctl alebo reštartovaním procesu servera.
