Éditer les vues, procédures stockées ou fonctions

Lors de l'édition du code d'une vue, d'une procédure stockée ou d'une fonction, au lieu d'utiliser la petite zone de texte de la boîte de dialogue des propriétés de l'objet, il peut être plus pratique d'utiliser les options de script de pgAdmin pour éditer l'objet dans l'outil de requête. Cela vous permet de bénéficier des fonctionnalités de l'éditeur de l'outil de requête telles que l'autocomplétion, la recherche et le remplacement, ainsi qu'un espace de travail plus grand.

Pour utiliser les options de script, dans la fenêtre de navigation de pgAdmin, faites un clic droit sur l'objet que vous souhaitez éditer, et sélectionnez l'option Script CREATE. Elle se trouve dans le sous-menu Scripts. Vous pouvez également sélectionner l'objet, puis choisir l'option Script CREATE dans le sous-menu Scripts du menu Outils.

Modifiez la définition de l'objet selon vos besoins, puis utilisez le bouton Exécuter la requête pour appliquer vos modifications. N'oubliez pas que la modification du nom d'une vue ou d'une fonction, ou de la signature d'une fonction ou procédure, peut entraîner la création d'un nouvel objet à côté de l'original. Cet effet secondaire utile permet de créer facilement de nouveaux objets similaires. Vous devrez peut-être actualiser la fenêtre de navigation de pgAdmin pour afficher les objets nouvellement créés.

Vous pouvez utiliser cette technique pour effectuer toutes sortes d'opérations sur différents objets, mais la syntaxe CREATE OR REPLACE de PostgreSQL la rend particulièrement utile pour éditer les vues, les procédures stockées et les fonctions.
