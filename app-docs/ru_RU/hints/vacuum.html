Рекомендуется выполнить VACUUM

Расчётное количество строк в таблице "" значительно отклоняется от фактического количества строк. Вам следует выполнить VACUUM ANALYZE для этой таблицы.

Вместо ручного выполнения команды VACUUM ANALYZE для этой таблицы (вы можете использовать меню обслуживания pgAdmin III для этого), следует рассмотреть регулярное или автоматическое выполнение VACUUM ANALYZE. Это можно сделать с помощью планировщика. PostgreSQL также предоставляет демон autovacuum, который отслеживает изменения, вносимые в базу данных, и автоматически выполняет команды VACUUM по мере необходимости. В большинстве случаев autovacuum будет наилучшим выбором.



Для чего нужен VACUUM?

Планировщик запросов PostgreSQL принимает решения на основе предположений, полученных из расчётного количества строк. Если фактическое количество строк слишком сильно отличается от расчётного, планировщик может принять неверное решение, что приведёт к неоптимальному плану запроса. Это может привести к низкой производительности.

Хранилищу PostgreSQL требуется VACUUM для фиксации идентификаторов транзакций в таблицах. Кроме того, устаревшие строки от обновлений и удалений не очищаются, пока команда VACUUM не будет выполнена для этой таблицы.
