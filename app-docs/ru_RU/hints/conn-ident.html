Сбой аутентификации Ident

Сервер не принимает текущего пользователя: сервер сообщает

Если появляется это сообщение, запись pg_hba.conf, найденная для вашей комбинации клиент / пользователь / база данных, имеет аутентификацию "ident". Некоторые дистрибутивы, например Debian, используют это по умолчанию. Для успешной аутентификации ident требуется дополнительная настройка; см. справку PostgreSQL. Для начинающих может быть более подходящим использовать другой метод аутентификации; пароли, зашифрованные MD5, являются хорошим выбором, который можно настроить записью в pg_hba.conf:

host all all 192.168.0.0/24 md5

Этот пример предоставляет доступ с паролем, зашифрованным MD5, ко всем базам данных всем пользователям в частной сети 192.168.0.0/24.

Вы можете использовать редактор pg_hba.conf, встроенный в pgAdmin III, для редактирования файла конфигурации pg_hba.conf. После изменения pg_hba.conf необходимо выполнить перезагрузку конфигурации сервера с помощью pg_ctl или остановки и перезапуска процесса сервера.
