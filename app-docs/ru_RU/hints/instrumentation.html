Инструментарий сервера

На сервере отсутствуют функции инструментария.

pgAdmin III использует некоторые вспомогательные функции, которые не доступны по умолчанию во всех версиях PostgreSQL. Они позволяют выполнять некоторые задачи, облегчающие работу с файлами журналов и конфигурации.

adminpack устанавливается и активируется по умолчанию, если вы используете 'официальный' дистрибутив pgInstaller PostgreSQL для Windows, и включён как модуль contrib во всех версиях PostgreSQL 8.2 и выше. Однако если вы используете любую другую версию PostgreSQL, вам потребуется установить его вручную. Для этого просто скопируйте соответствующий исходный код adminpack в каталог /contrib вашего предварительно настроенного дерева исходных кодов PostgreSQL. Вы можете скачать adminpack здесь.

Выполните следующие команды (заменив admin на admin81 если необходимо):

    # cd $PGSRC/contrib/admin
    # make all
    # make install

Теперь модуль должен быть собран и установлен в вашу существующую установку PostgreSQL. Чтобы использовать его, необходимо установить модуль в базу данных, которую вы используете как Служебную базу данных pgAdmin (обычно postgres, хотя вы можете использовать другую базу данных). Для этого найдите файл admin.sql или admin81.sql, установленный на предыдущем шаге — обычно он находится в /usr/local/pgsql/share/contrib/ — и выполните его для вашей базы данных. Обычно это нужно делать как суперпользователь PostgreSQL:

    $ psql postgres < /usr/local/pgsql/share/contrib/admin.sql

При следующем подключении к серверу с помощью pgAdmin он обнаружит наличие adminpack и предложит полную функциональность.

Расширенный инструментарий сервера не поддерживается для PostgreSQL 7.3 и 7.4.
