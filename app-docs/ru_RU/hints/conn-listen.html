Сервер не прослушивает

Сервер не принимает соединения: библиотека соединения сообщает

Если вы столкнулись с этим сообщением, проверьте, действительно ли сервер, к которому вы пытаетесь подключиться, запущен с PostgreSQL на указанном порту. Проверьте наличие сетевого соединения от клиента к серверу с помощью ping или аналогичных средств. Правильно ли настроены сеть / VPN / SSH-туннель / брандмауэр?

По соображениям безопасности PostgreSQL не прослушивает все доступные IP-адреса на сервере изначально. Для доступа к серверу по сети необходимо сначала включить прослушивание на нужном адресе.

Для серверов PostgreSQL начиная с версии 8.0 это управляется параметром "listen_addresses" в файле postgresql.conf. Здесь можно ввести список IP-адресов, на которых сервер должен прослушивать, или просто использовать '*' для прослушивания всех доступных IP-адресов. Для более старых серверов (версии 7.3 или 7.4) необходимо установить параметр "tcpip_socket" в 'true'.

Вы можете использовать редактор postgresql.conf, встроенный в pgAdmin III, для редактирования файла конфигурации postgresql.conf. После изменения этого файла необходимо перезапустить процесс сервера, чтобы настройка вступила в силу.

Если вы перепроверили конфигурацию, но всё ещё получаете это сообщение об ошибке, маловероятно, что вы столкнулись с критической ошибкой PostgreSQL. Вероятно, у вас проблемы с сетевым подключением низкого уровня (например, конфигурация брандмауэра). Пожалуйста, тщательно проверьте это, прежде чем сообщать об ошибке сообществу PostgreSQL.
