Кодировка базы данных

База данных создана для хранения данных в кодировке SQL_ASCII. Эта кодировка определена только для 7-битных символов; значение символов с установленным 8-м битом (не-ASCII символы 127-255) не определено. Следовательно, сервер не может конвертировать данные в другие кодировки.

Если вы храните не-ASCII данные в базе данных, настоятельно рекомендуется использовать подходящую кодировку, представляющую вашу локальную символьную таблицу, чтобы воспользоваться автоматической конвертацией в разные клиентские кодировки при необходимости. Если вы храните не-ASCII данные в базе SQL_ASCII, вы можете столкнуться со странными символами, записываемыми или читаемыми из базы данных, вызванными проблемами конвертации кодировок. Это может доставить много хлопот при доступе к базе данных из разных клиентских программ и драйверов.

Для большинства установок кодировка Unicode (UTF8) предоставит наиболее гибкие возможности.
