Рекомендуется включить autovacuum

Начиная с PostgreSQL 8.1, процесс autovacuum, ранее реализованный как внешняя служба, теперь интегрирован в сервер. Он будет автоматически запускать VACUUM, если изменилось настраиваемое количество данных.

Выполнение VACUUM обязательно на сервере PostgreSQL для поддержания согласованности данных и производительности сервера. Для начала рекомендуется включить демон autovacuum, установив 'autovacuum', 'stats_start_collector' и 'stats_row_level' в postgresql.conf в 'on'.

В PostgreSQL 8.3 и новее autovacuum включён по умолчанию, если только вы не отключили 'track_activities' или 'track_counts'.

Ценой обычно незначительного снижения производительности из-за отслеживания изменений данных вы получите автоматическое выполнение VACUUM.

Чтобы настроить производительность autovacuum для индивидуальных потребностей, возможно, потребуется отрегулировать его параметры с помощью глобальных параметров в postgresql.conf или индивидуально для каждой таблицы; см. документацию.
