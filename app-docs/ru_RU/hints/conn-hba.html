Доступ к базе данных запрещён

Сервер не предоставляет доступ к базе данных: сервер сообщает

Для доступа к базе данных на сервере PostgreSQL необходимо сначала предоставить базовый доступ к серверу для вашего клиента (Host Based Authentication). PostgreSQL проверит файл pg_hba.conf на наличие записи, соответствующей адресу клиента / имени пользователя / базе данных, прежде чем будут оценены списки контроля доступа SQL GRANT.

Начальные настройки в pg_hba.conf довольно ограничительны, чтобы избежать нежелательных брешей в безопасности, вызванных нерассмотренными, но обязательными системными настройками. Вероятно, вы захотите добавить что-то вроде

host all all 192.168.0.0/24 md5

Этот пример предоставляет доступ с паролем, зашифрованным MD5, ко всем базам данных всем пользователям в частной сети 192.168.0.0/24.

Вы можете использовать редактор pg_hba.conf, встроенный в pgAdmin III, для редактирования файла конфигурации pg_hba.conf. После изменения pg_hba.conf необходимо выполнить перезагрузку конфигурации сервера с помощью pg_ctl или остановки и перезапуска процесса сервера.
