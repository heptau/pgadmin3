Instrumentacja serwera

Serwerowi brakuje funkcji instrumentacyjnych.

pgAdmin III używa pewnych funkcji pomocniczych, które nie są domyślnie dostępne we wszystkich wersjach PostgreSQL. Umożliwiają one wykonanie niektórych zadań, które ułatwiają pracę z plikami dziennika i plikami konfiguracyjnymi.

adminpack jest instalowany i aktywowany domyślnie, jeśli używasz 'oficjalnej' dystrybucji pgInstaller PostgreSQL dla Windows i jest dołączony jako moduł contrib we wszystkich wersjach PostgreSQL 8.2 i wyższych. Jeśli jednak używasz innej wersji PostgreSQL, będziesz musiał zainstalować go ręcznie. Aby to zrobić, po prostu skopiuj odpowiedni kod źródłowy adminpack do katalogu /contrib swojego skonfigurowanego drzewa źródłowego PostgreSQL. Możesz pobrać adminpacki tutaj.

Uruchom następujące polecenia (zastępując admin na admin81 jeśli to konieczne):

    # cd $PGSRC/contrib/admin
    # make all
    # make install

Moduł powinien być teraz zbudowany i zainstalowany w Twojej istniejącej instalacji PostgreSQL. Aby go użyć, musisz teraz zainstalować moduł w bazie danych, której używasz jako Baza konserwacyjna pgAdmin (normalnie postgres, choć możesz użyć innej bazy danych). Aby to zrobić, zlokalizuj plik admin.sql lub admin81.sql zainstalowany w poprzednim kroku - zwykle można go znaleźć w /usr/local/pgsql/share/contrib/ i uruchom go względem swojej bazy danych. Zwykle musisz to zrobić jako superużytkownik PostgreSQL:

    $ psql postgres < /usr/local/pgsql/share/contrib/admin.sql

Przy następnym połączeniu z serwerem za pomocą pgAdmin, wykryje on obecność adminpack i zaoferuje pełną funkcjonalność.

Rozszerzona instrumentacja serwera nie jest obsługiwana dla PostgreSQL 7.3 i 7.4.
