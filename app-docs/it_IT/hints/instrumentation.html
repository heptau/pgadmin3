Strumentazione del server

Al server mancano funzioni di strumentazione.

pgAdmin III utilizza alcune funzioni di supporto che non sono disponibili per impostazione predefinita in tutte le versioni di PostgreSQL. Queste consentono alcune attività che semplificano la gestione dei file di log e dei file di configurazione.

adminpack viene installato e attivato per impostazione predefinita se si utilizza la distribuzione 'ufficiale' di pgInstaller di PostgreSQL per Windows ed è incluso come modulo contrib in tutte le versioni di PostgreSQL 8.2 e successive. Tuttavia, se si utilizza qualsiasi altra versione di PostgreSQL, sarà necessario installarlo manualmente. Per farlo, basta copiare il codice sorgente appropriato di adminpack nella directory /contrib del proprio albero del codice sorgente di PostgreSQL preconfigurato. Puoi scaricare gli adminpack qui.

Eseguire i seguenti comandi (sostituendo admin con admin81 se necessario):

    # cd $PGSRC/contrib/admin
    # make all
    # make install

Il modulo ora dovrebbe essere compilato e installato nella tua installazione esistente di PostgreSQL. Per essere utilizzato, devi ora installare il modulo nel database che usi come Database di Manutenzione di pgAdmin (normalmente postgres, anche se potresti voler usare un database diverso). Per farlo, individua il file admin.sql o admin81.sql installato nel passaggio precedente - normalmente si trova in /usr/local/pgsql/share/contrib/ ed eseguilo sul database. Di solito è necessario farlo come superutente PostgreSQL:

    $ psql postgres < /usr/local/pgsql/share/contrib/admin.sql

La prossima volta che ti connetterai al server con pgAdmin, rileverà la presenza di adminpack e offrirà funzionalità complete.

La strumentazione estesa del server non è supportata per PostgreSQL 7.3 e 7.4.
