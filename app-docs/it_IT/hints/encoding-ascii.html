Codifica del database

Il database è stato creato per memorizzare i dati utilizzando la codifica SQL_ASCII. Questa codifica è definita solo per caratteri a 7 bit; il significato dei caratteri con l'8° bit impostato (caratteri non ASCII 127-255) non è definito. Di conseguenza, non è possibile per il server convertire i dati in altre codifiche.

Se si memorizzano dati non ASCII nel database, si consiglia vivamente di utilizzare una codifica adeguata che rappresenti il proprio set di caratteri locali per beneficiare della conversione automatica verso diverse codifiche client quando necessario. Se si memorizzano dati non ASCII in un database SQL_ASCII, si potrebbero incontrare caratteri strani scritti o letti dal database, causati da problemi di conversione del codice. Ciò potrebbe causare molti problemi durante l'accesso al database utilizzando diversi programmi e driver client.

Per la maggior parte delle installazioni, la codifica Unicode (UTF8) fornirà le capacità più flessibili.
