Toegang tot database geweigerd

De server geeft geen toegang tot de database: de server meldt

Om toegang te krijgen tot een database op een PostgreSQL-server, moet u eerst basis toegang tot de server verlenen voor uw client (Host Based Authentication). PostgreSQL controleert het pg_hba.conf-bestand of er een patroon is dat overeenkomt met uw clientadres / gebruikersnaam / database en ingeschakeld is voordat SQL GRANT-toegangscontrolelijsten worden geëvalueerd.

De initiële instellingen in pg_hba.conf zijn vrij restrictief om ongewenste beveiligingslekken te voorkomen die worden veroorzaakt door niet-beoordeelde maar verplichte systeeminstellingen. U wilt waarschijnlijk iets toevoegen zoals

host all all 192.168.0.0/24 md5

Dit voorbeeld verleent MD5-versleutelde wachtwoordtoegang tot alle databases voor alle gebruikers in het privénetwerk 192.168.0.0/24.

U kunt de pg_hba.conf-editor gebruiken die is ingebouwd in pgAdmin III om het configuratiebestand pg_hba.conf te bewerken. Na het wijzigen van pg_hba.conf moet u een herlading van de serverconfiguratie activeren met pg_ctl of door het serverproces te stoppen en opnieuw te starten.
