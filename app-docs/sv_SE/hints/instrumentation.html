Serverinstrumentering

Servern saknar instrumenteringsfunktioner.

pgAdmin III använder några stödfunktioner som inte är tillgängliga som standard i alla PostgreSQL-versioner. Dessa möjliggör vissa uppgifter som gör livet enklare när man hanterar loggfiler och konfigurationsfiler.

adminpack installeras och aktiveras som standard om du använder den 'officiella' pgInstaller-distributionen av PostgreSQL för Windows och ingår som en contrib-modul i alla versioner av PostgreSQL 8.2 och senare. Om du dock använder någon annan version av PostgreSQL måste du installera det manuellt. För att göra detta, kopiera helt enkelt lämplig adminpack-källkod till /contrib-katalogen i ditt förkonfigurerade PostgreSQL-källkodsträd. Du kan ladda ner adminpacken här.

Kör följande kommandon (ersätt admin med admin81 om det behövs):

    # cd $PGSRC/contrib/admin
    # make all
    # make install

Modulen bör nu vara byggd och installerad i din befintliga PostgreSQL-installation. För att använda den måste du nu installera modulen i databasen du använder som pgAdmins Underhållsdatabas (normalt postgres, även om du kanske vill använda en annan databas). För att göra detta, lokalisera admin.sql eller admin81.sql-filen som installerades i föregående steg - normalt finns den i /usr/local/pgsql/share/contrib/ och kör den mot din databas. Du behöver normalt göra detta som PostgreSQL-superanvändare:

    $ psql postgres < /usr/local/pgsql/share/contrib/admin.sql

Nästa gång du ansluter till servern med pgAdmin kommer den att upptäcka närvaron av adminpack och erbjuda full funktionalitet.

Utökad serverinstrumentering stöds inte för PostgreSQL 7.3 och 7.4.
