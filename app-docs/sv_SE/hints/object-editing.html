Redigera vyer, lagrade procedurer och funktioner

När du redigerar koden för en vy, lagrad procedur eller funktion, istället för att använda den lilla textrutan i egenskapsdialogrutan för objektet, kan du tycka att det är bekvämare att använda skriptalternativen i pgAdmin för att redigera objektet i frågeverktyget. Detta ger dig fördelarna med frågeverktygets redigeringsfunktioner som autokomplettering, sök och ersätt samt större arbetsyta.

För att använda skriptalternativen, i pgAdmin-webbläsarfönstret, högerklicka på objektet du vill redigera och välj alternativet CREATE-skript. Det kan finnas på undermenyn Skript. Alternativt kan du välja objektet och sedan välja CREATE-skript-alternativet från undermenyn Skript i menyn Verktyg.

Redigera objektets definition efter behov och använd knappen Kör fråga för att tillämpa dina ändringar. Glöm inte att ändra ett vy- eller funktionsnamn, eller signaturen för en funktion eller procedur kan resultera i att ett nytt objekt skapas bredvid det ursprungliga. Denna användbara bieffekt gör det bekvämt att skapa nya, men liknande objekt. Du kan behöva uppdatera pgAdmin-webbläsaren för att visa nyskapade objekt.

Du kan använda denna teknik för att utföra alla typer av operationer på olika objekt, men PostgreSQLs CREATE OR REPLACE-syntax gör den särskilt användbar för att redigera vyer, lagrade procedurer och funktioner.
