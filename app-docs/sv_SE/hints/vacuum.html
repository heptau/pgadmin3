VACUUM rekommenderas

Det uppskattade radantalet i tabellen "" avviker betydligt från det verkliga radantalet. Du bör köra VACUUM ANALYZE på denna tabell.

Istället för att utfärda ett manuellt VACUUM ANALYZE-kommando på denna tabell (du kan använda pgAdmin III:s underhållsmeny för detta), bör du överväga att köra VACUUM ANALYZE regelbundet eller automatiskt. Detta kan uppnås med en schemaläggare. PostgreSQL tillhandahåller också autovacuum-demonen, som spårar ändringar som görs i databasen och utfärdar vacuum-kommandon automatiskt vid behov. I de flesta fall kommer autovacuum att vara det bästa valet.



Vad är VACUUM bra för?

PostgreSQLs frågeplanerare fattar sina beslut baserat på antaganden från det uppskattade radantalet. Om det verkliga radantalet skiljer sig för mycket från det uppskattade radantalet kan planeraren fatta ett felaktigt beslut, vilket resulterar i en frågeplan som inte är optimal. Detta kan leda till dålig prestanda.

PostgreSQLs lagring behöver VACUUM för att transaktions-ID:n i tabeller ska fixeras. Dessutom rensas inte föråldrade rader från uppdateringar och borttagningar förrän VACUUM-kommandot har utfärdats på den tabellen. Djupgående information finns i onlinedokumentationen, tryck bara på hjälpknappen.
