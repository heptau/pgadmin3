Ident-autentisering misslyckades

Servern accepterar inte den aktuella användaren: servern rapporterar

Om detta meddelande visas är pg_hba.conf-posten som hittades för din klient / användare / databaskombination inställd på "ident"-autentisering. Vissa distributioner, t.ex. Debian, har detta som standard. För att utföra ident-baserad autentisering framgångsrikt krävs ytterligare konfiguration; se PostgreSQL-hjälpen för detta. För en nybörjare kan det vara lämpligare att använda en annan autentiseringsmetod; MD5-krypterade lösenord är ett bra val, som kan konfigureras med en post i pg_hba.conf så här:

host all all 192.168.0.0/24 md5

Detta exempel beviljar MD5-krypterad lösenordsåtkomst till alla databaser för alla användare i det privata nätverket 192.168.0.0/24.

Du kan använda pg_hba.conf-redigeraren som är inbyggd i pgAdmin III för att redigera konfigurationsfilen pg_hba.conf. Efter att ha ändrat pg_hba.conf måste du utlösa en omladdning av serverkonfigurationen med pg_ctl eller genom att stoppa och starta om serverprocessen.
